# o3NatSrv
内网渗透服务端
=======

公元2018年12月18日起，OpenShift3 的免费服务将不能再永久使用，一个服务只能使用60天。想用 OpenShift3 维持服务的运行已不再可靠。现决定用 OpenShift3 的Web服务做公网代理，来访问我内网主机部署的服务。

*******************************************************************

缓存：
-------------------------------------------------------------------

- 简单的内网渗透，接收拆包后的数据依然存在顺序问题。
- 并发测试
- O3上的测试
	- O3中间有转发。客户端断开连接时，不会触发服务端的end事件。故需要给服务端发送一个断开指令。
	- O3 有 timeout 时限，连接无法长时间挂机！时限约25秒左右。
- WebSocket连接测试

*******************************************************************

计划：
-------------------------------------------------------------------

- 实现内网渗透
- OpenShift3 免费服务规定：
	1. 每72小时必须休息18小时
	1. 每30分钟不活动将会自动休眠
	1. 60天后服务自动过期
	故，为保证服务的稳定性，需考虑服务的多班倒。
